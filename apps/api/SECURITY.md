# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability, please report it privately.

- Contact: `wrobb@vergoltd.com`
- Include: a clear description, affected endpoint(s)/file(s), steps to reproduce, and impact.
- Please do not open a public issue with exploit details.

We aim to acknowledge reports within 3 business days.

## Sensitive Data

When reporting, do not include:
- Passwords, reset links, access tokens, API keys, or private keys
- Full database connection strings

Redact secrets and provide placeholders where possible.
